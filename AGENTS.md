# AI Agent Communication & Development Guidelines

このファイルは、本プロジェクトにおいてAIエージェント（Antigravity等）と協力して開発を行う際のルールとナレッジをまとめたものです。

## 1. コミュニケーションルール

- **言語設定**: すべての会話と作成されるドキュメントは**日本語**で行う。
- **提案ベース**: ユーザーの指示を待つだけでなく、より良いアーキテクチャや技術選定があれば積極的に提案する（例：GoからPython/Streamlitへの変更提案）。
- **ステップバイステップ**: 一度に全てを行わず、実装→検証→フィードバックのサイクルを回す。

## 2. 開発プロセス

- **計画・タスク管理**:
    - GitHub Issueを活用し、タスクを適切に細分化して管理する（巨大なIssueは避ける）。
    - 完了したタスクはクローズし、進捗を明確にする。
- **ブランチ戦略**:
    - `main` ブランチへの直接コミットは原則避け、機能ごとにFeatureブランチを切る（例: `feature/python-streamlit`）。
    - 作業完了後、GitHub CLI (`gh`) を使用してPull Requestを作成する。
- **ファイル操作**:
    - ファイル編集時は、単純な置換だけでなく、文脈を考慮した修正を行う。
    - 既存のファイルを読み込む際は、ツールを使用して中身と行数を確認してから編集する。

- Issue/PRは日本語で記載する
- mainブランチに戻ったらgit pullしてmainを最新化する

## 3. 技術的ナレッジ
- **uv**: 高速なPythonパッケージマネージャー。`pip`や`venv`の代わりに`uv sync`, `uv run`を使用する。
- **Ruff**: 高速なリンター/フォーマッター。`make lint` / `make format` で実行。
- **Streamlit**: WebUIフレームワーク。

## 4. プロジェクト構成
- **Python環境**: `uv` + `pyproject.toml` を基本とする。
- **ディレクトリ構造**:
    - `app.py`: UIエントリポイント
    - `src/`: ビジネスロジック、DBアクセス層
